## Overview

The `Joystick` class provides a high-level interface for reading the analog joystick on the BOOSTXL-EDUMKII BoosterPack.

It supports:

- X/Y analog readings
- center push button handling (inherits from `Button`)
- 8-way direction quantization (`direction8()`)
- configurable deadzone, smoothing, and hysteresis
- callbacks for movement and direction events

## Hardware Connections

| Function | MCU Pin | Port  | Description                  |
| :------- | :------ | :---- | :--------------------------- |
| X-Axis   | PD2     | GPIOD | Joystick horizontal movement |
| Y-Axis   | PK1     | GPIOK | Joystick vertical movement   |
| Button   | PA0     | GPIOA | Joystick center push button  |

## Quick Start

<Steps>

1. Construct `Joystick js(JSX, JSY, JS1);`
2. Call `js.begin();`
3. Optionally call `js.calibrateCenter(32);`
4. In a periodic loop/task, call `js.tick();`
5. Read `js.x()`, `js.y()`, or `js.direction8()`

</Steps>

```cpp
#include "joystick.h"

Joystick js(JSX, JSY, JS1);

int main(void) {
    uint32_t sysclk = SysCtlClockFreqSet(
        SYSCTL_XTAL_25MHZ | SYSCTL_OSC_MAIN |
        SYSCTL_USE_PLL | SYSCTL_CFG_VCO_480,
        120000000
    );
    (void)sysclk;

    js.begin();
    js.calibrateCenter(32);

    while (1) {
        js.tick();

        JoystickDir dir = js.direction8();
        if (dir == JoystickDir::E) {
            // moved right
        } else if (dir == JoystickDir::W) {
            // moved left
        }

        SysCtlDelay((120000000 / 100) / 3); // ~10 ms
    }
}
```

## API Reference (Most Used)

| Method | Purpose |
| :----- | :------ |
| `begin()` | Initializes ADC/GPIO and button support. |
| `tick()` | Updates readings, direction state, and callbacks. Call periodically. |
| `x()`, `y()` | Returns normalized axis values in `[-1, 1]`. |
| `magnitude()` | Returns radial magnitude in `[0, 1]`. |
| `angleDeg()` | Returns joystick angle in degrees. |
| `direction8()` | Returns quantized direction (`N`, `NE`, `E`, `SE`, `S`, `SW`, `W`, `NW`, `Center`). |

---

## Configuration Parameters

| Method | Description |
| :----- | :---------- |
| `setDeadzone(float dz)` | Sets neutral zone radius (`0..1`). |
| `setSmoothingAlpha(float a)` | Sets IIR smoothing (`0..1`). |
| `setDirectionThreshold(float magMin)` | Magnitude needed to enter a direction. |
| `setDirectionHysteresis(float magBack, float degBack)` | Magnitude to return to center (hysteresis). |
| `setMoveEpsilon(float eps)` | Minimum normalized movement to emit move events. |
| `setRepeatIntervalMs(uint32_t ms)` | Repeat period while held in a direction. |
| `setTickIntervalMs(uint32_t ms)` | Internal tick timebase; match your real polling period. |
| `calibrateCenter(uint16_t samples)` | Averages centered joystick samples at startup. |

Suggested baseline configuration:

```cpp
js.setTickIntervalMs(10);
js.setDeadzone(0.10f);
js.setSmoothingAlpha(0.20f);
js.setDirectionThreshold(0.30f);
js.setDirectionHysteresis(0.22f, 15.0f);
js.setRepeatIntervalMs(150);
```

## Events

| Event | Signature | Trigger |
| :---- | :-------- | :------ |
| `onMove` | `void fn(Joystick&)` | Movement exceeds epsilon. |
| `onTiltStart` | `void fn(Joystick&)` | Exits center/deadzone. |
| `onTiltStop` | `void fn(Joystick&)` | Returns to center/deadzone. |
| `onDirectionChanged` | `void fn(Joystick&)` | Quantized direction changes. |
| `onRepeat` | `void fn(Joystick&)` | Repeats while held away from center. |
| `onCenter` | `void fn(Joystick&)` | Emits when centered after tilt. |

Example callback registration:

```cpp
static void OnDirectionChanged(Joystick& js) {
    JoystickDir d = js.direction8();
    // handle direction
}

js.onDirectionChanged(&OnDirectionChanged);
js.attachClick([](){
    // center button click
});
```

## General Pattern: One Action per Deflection

This pattern is useful for any menu/UI navigation where you want one action per intentional move.

```cpp
bool readyForNextMove = true;

for (;;) {
    js.tick();
    JoystickDir d = js.direction8();

    if (d == JoystickDir::Center) {
        readyForNextMove = true;
    } else if (readyForNextMove && d == JoystickDir::E) {
        menuIndex = (menuIndex + 1u < MENU_ITEMS) ? (menuIndex + 1u) : 0u;
        readyForNextMove = false;
    } else if (readyForNextMove && d == JoystickDir::W) {
        menuIndex = (menuIndex > 0u) ? (menuIndex - 1u) : (MENU_ITEMS - 1u);
        readyForNextMove = false;
    }

    vTaskDelay(pdMS_TO_TICKS(30));
}
```

## General Pattern: Continuous Analog Control

Use normalized X and Y values for smooth control (cursor, camera, PWM setpoint, etc.).

```cpp
for (;;) {
    js.tick();

    float x = js.x();   // [-1, 1]
    float y = js.y();   // [-1, 1]

    // Example: map X to a signed command in [-100, 100]
    int command = (int)(x * 100.0f);
    ApplyCommand(command);

    // Example: only act on strong upward movement
    if (y > 0.7f) {
        TriggerAction();
    }

    vTaskDelay(pdMS_TO_TICKS(10));
}
```

## General Pattern: Event-Driven Direction Handling

Use callbacks when you prefer event-style code over polling checks.

```cpp
static void OnDirectionChanged(Joystick& js) {
    switch (js.direction8()) {
        case JoystickDir::N: MoveUp(); break;
        case JoystickDir::S: MoveDown(); break;
        case JoystickDir::E: MoveRight(); break;
        case JoystickDir::W: MoveLeft(); break;
        default: break;
    }
}

int main(void) {
    Joystick js(JSX, JSY, JS1);
    js.begin();
    js.calibrateCenter(32);
    js.onDirectionChanged(&OnDirectionChanged);

    while (1) {
        js.tick();
        SysCtlDelay((120000000 / 100) / 3); // ~10 ms
    }
}
```

## Troubleshooting

- No movement detected:
Check pin mapping (`JSX`, `JSY`, `JS1`) and confirm ADC-capable pins.
- Direction chatters near center:
Increase deadzone or smoothing.
- Left/right feels swapped:
Use `setSwapXY(true)` for board variants and/or `setInvertX(true)`.
- Too many repeated events:
Increase `setRepeatIntervalMs(...)` or use the one-move-per-tilt pattern.
